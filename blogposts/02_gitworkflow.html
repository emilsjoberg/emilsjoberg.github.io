Git/Github workflow for a two person team, or when using two computers

May 2013

Countless people have posted Git and Gthub tutorials. I never thought I would do the same. When I started using Github, I wanted a tutorial which would walk me through pushing files into Git and Github, then successfully pull those files onto another computer. I could not find one. I was the typical, frusterated, first timer at 2am. I thought I would post one for myself and others. I assume you have a Github account. Create a new repository (my username is dfeusse. Subsitute your username for mine). Make sure you have generated SSH keys. If this is unfamilar, follow the directions in this link. It is very simple. Navigate to the desired directory, then follow these commands.

$ git init
$ got config core.autocrlf false
$ git add .
$ git commit -m "initial commit"
$ git remote add origin git@github.com:dfeusse/x.git
$ git pull origin master (only use this if you created a readme file when making your repository)
$ git push origin master

Your files are now on Github. The second team member, or on your second computer, you can pull the files. Follow these commands after navigating into the correct directory.

$ git init
$ git remote add origin git@github.com:dfeusse/x.git
$ git pull origin master

Now, make changes, push to Github, repeat.
